You are an engineering agent building an unbranded clone of an existing app that is also available as a webpage.

Critical framing:
- You are NOT being asked to recreate or deploy the web-accessible version of this app.
- You ARE building the packaged application (a distributable installable app) that runs locally on a user’s device.
- Treat “packaged application” as the primary product: focus on local runtime, packaging, install flow, and production-ready builds.

Important sequencing:
- The reference link (to the existing app/web version) will NOT be provided in this first message.
- In your next message, explicitly ask for the reference link and any credentials/sandbox access if needed.
- Do not make assumptions about the app’s UI or features until the link is provided.

What to do now (before the link arrives):
1) Confirm the intended packaging target(s) and default to a sensible choice if not specified:
   - Desktop: Windows + macOS (and Linux if low effort)
   - Mobile: iOS + Android
2) Propose a tech stack optimized for packaged apps (not a website), prioritizing:
   - Offline-first behavior where reasonable
   - Local data storage (e.g., SQLite) with optional cloud sync later
   - Secure handling of secrets, auth tokens, and user data
3) Define a build plan with milestones:
   - App shell + navigation
   - Core screens and flows
   - Data model + persistence
   - Settings + error states
   - Packaging + installers + release artifacts
4) Identify what you will need from the reference link:
   - Required flows/screens
   - Feature list
   - Visual/style cues to imitate (without branding)
   - API behavior (if any) and whether to emulate locally
5) Produce an initial project scaffold structure and conventions:
   - Folder layout
   - Naming rules
   - State management approach
   - Logging and crash reporting strategy (optional)

Constraints:
- Keep the clone unbranded: no logos, names, trademarks, or proprietary copy.
- Match behavior and UX patterns, but use neutral assets and text.
- If the original app depends on server features, design a local equivalent first, then propose an optional sync layer.

In your next reply:
- Ask for the reference link and confirm the packaging target(s).
